# Roshambo (Godot) Agent

本文档适用于 `godot/` 子工程的开发协作，约束 Agent 在 Godot + C# 的最佳实践下工作。

## 首次对话前置
- 若尚未阅读过仓库根目录 `AGENTS.md`，开始任何修改前先阅读它，继承其中的通用身份/纪律/文档规则。

## 技术栈与边界
- 引擎：Godot（以项目内版本与设置为准）
- 语言：C#
- 本子工程与 `ts/` 相互独立：除非用户明确要求，否则不要在一次变更里同时改两边

## 代码风格与工程约定
- 采用清晰的命名与目录边界（Scene / Script / Resources 分离）
- C# 使用 PascalCase（类型/方法/属性），私有字段使用 `_camelCase`
- 避免把游戏规则散落在节点脚本里：优先提取到纯逻辑类/资源类

## Godot + C# 最佳实践
- 信号（Signals）优先于硬引用：用事件/信号解耦 UI 与逻辑
- 节点生命周期：在 `_Ready()` 中绑定引用与信号；在 `_ExitTree()`/`Dispose` 做清理
- 性能：避免每帧 `GetNode()`；缓存节点引用；避免频繁分配临时对象
- 资源：使用 `Resource`/`PackedScene` 管理可配置数据；必要时引入 Scriptable-like 的资源定义

## 变更纪律
- 开始前确认当前改动只涉及 `godot/`
- 每次“明确可验证的修改”完成后：
  - 运行一次可验证的导出/运行检查（以用户要求为准）
  - 在仓库根目录 `PROGRESS.md` 的“最近完成”追加一条并附 ISO 8601 时间戳
